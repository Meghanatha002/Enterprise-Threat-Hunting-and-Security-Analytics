# Insider Access Anomaly Detection Using Security Analytics

## Professional Summary

This case study demonstrates the application of security analytics techniques to identify anomalous user access behavior that may indicate insider threats, compromised accounts, or unauthorized access attempts. By analyzing user activity patterns, the investigation highlights how statistical analysis can assist Security Operations Center (SOC) analysts in prioritizing suspicious events for further investigation.

---

# Project Overview

Insider threats remain one of the most challenging cybersecurity risks because malicious or compromised users often operate with legitimate credentials. This project analyzes user access activity to identify abnormal behavior that deviates from expected usage patterns.

The investigation demonstrates how data-driven analytics can support threat hunting and insider threat detection by highlighting suspicious user activity that warrants additional review.

---

# Objectives

The primary objectives of this project were to:

- Analyze user access activity using Python.
- Identify anomalous user behavior through statistical analysis.
- Detect unusual access frequencies and activity patterns.
- Demonstrate how security analytics supports insider threat investigations.
- Develop a repeatable analytical workflow for enterprise security monitoring.

# Methodology

The investigation followed the workflow below:

1. Imported user access data into Python.
2. Performed data cleaning and preprocessing.
3. Analyzed user access frequency and activity patterns.
4. Identified anomalous user behavior based on statistical characteristics.
5. Prioritized suspicious accounts for additional investigation.
6. Documented findings and analytical observations.
