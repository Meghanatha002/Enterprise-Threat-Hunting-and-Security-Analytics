# Domain Frequency Analysis for Threat Hunting and Suspicious Domain Identification

## Professional Summary

This case study demonstrates a frequency-based analytical approach for identifying uncommon domain names that may warrant further security investigation. Using statistical character frequency analysis, domain labels were evaluated to identify lexical anomalies that can support threat hunting activities and the early detection of suspicious domains. Although this technique alone cannot determine whether a domain is malicious, it provides an effective preprocessing method for prioritizing domains for additional investigation.

---

# Project Overview

Domain names generated by malicious infrastructure often exhibit statistical characteristics that differ from legitimate domains. This project applies frequency-based scoring to domain labels in order to identify uncommon naming patterns that may indicate suspicious or algorithmically generated domains.

The implementation demonstrates how statistical analysis can be integrated into a cybersecurity workflow to reduce large domain datasets into a manageable set of candidates for further investigation.

---

# Objectives

The primary objectives of this project were to:

* Analyze a ranked domain dataset using Python.
* Extract primary domain labels using the **tldextract** library.
* Apply a character frequency scoring algorithm to each unique domain label.
* Identify domains with scores below the selected threshold.
* Demonstrate how statistical domain analysis can support cybersecurity investigations and threat hunting activities.

# Methodology

The analysis followed the workflow below:

1. Loaded a ranked domain dataset into Python.
2. Extracted the primary domain label while removing subdomains and top-level domains.
3. Applied a statistical character frequency scoring algorithm to each unique domain label.
4. Calculated a cumulative frequency score for every domain.
5. Identified domain labels with scores below the predefined threshold (3.5).
6. Removed duplicate labels to generate the final list of unique suspicious candidates.

This workflow demonstrates a practical data preprocessing technique commonly used during DNS analytics and threat hunting.

# Key Findings

* Successfully processed a large ranked domain dataset.
* Extracted unique domain labels for statistical analysis.
* Applied frequency-based scoring to every domain label.
* Identified **32,898** unique domain labels below the selected threshold.
* Demonstrated how statistical analysis can prioritize suspicious domains for additional investigation.

# Conclusion

This project demonstrates how frequency-based domain analysis can be incorporated into enterprise threat hunting workflows to identify uncommon domain names for further investigation. While frequency scoring should not be used as a standalone detection mechanism, it provides an efficient statistical filtering technique that can significantly reduce investigation scope when combined with DNS telemetry, threat intelligence, and behavioral analysis.

The techniques demonstrated in this project establish a foundation for more advanced domain-based detection methods, including DNS analytics, Domain Generation Algorithm (DGA) detection, and machine learning–based threat classification.
